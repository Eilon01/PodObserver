Welcome to Pod Observer

About Pod Observer

Pod Observer is a Kubernetes-based tool that helps you monitor and manage your pods effectively. With a user-friendly interface and a set of essential commands, Pod Observer provides real-time information on pod health, logs, and service versions.

How It Works

Pod Observer integrates with the Kubernetes API to gather information about running pods, their logs, and the services they provide. Whether you are deploying a new application or maintaining an existing one, Pod Observer helps you stay informed and troubleshoot issues quickly.

Available Commands

  • /help - This command provides information on how to use the bot and its available commands.
  • /get-pods - Lists all the running pods in the Kubernetes cluster, their uptime, and the version of each service. This command helps you monitor the status of your services in real-time.
  • /get-logs <pod-name> <n> - Retrieves the last n log lines from the specified pod. You can use this command to quickly check logs for troubleshooting issues or verifying system behavior.
    Example: /get-logs my-pod-1 50 will return the last 50 log lines from the pod named my-pod-1.
